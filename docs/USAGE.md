# Usage

## Basic pattern

Give Terminator an objective:

```text
Use Terminator to <goal>
```

## Examples

### Project recovery

```text
Use Terminator to inspect this repository, identify the main blocker, fix safe issues, and verify the project.
```

### Research and paper drafting

```text
Use Terminator to research this topic, organize reliable sources, and create a structured paper draft.
```

### Presentation creation

```text
Use Terminator to create a professional presentation from the files in this folder.
```

### Curriculum generation

```text
Use Terminator to create a professional CV using the documents in these folders as sources.
```

### Readiness check

```text
Terminator
```

Terminator scans the current context and asks for the objective.

## Good objectives

Strong objectives include:

- the desired result
- input sources
- output format
- constraints
- quality requirements
- verification expectation

Example:

```text
Use Terminator to create a 10-slide professional presentation about quantum computing using the PDFs in ./sources, export it as Markdown, and verify that every slide has a clear title and source reference.
```
